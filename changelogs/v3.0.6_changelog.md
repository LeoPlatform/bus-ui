# Botmon v3.0.6 Changelog

## Improvements

We have added some needed improvements to Botmon, which are detailed below. 

As always if there are questions/issues using Botmon or if you have improvements you would like made reach out to [#h_rstreams](https://chubrox.slack.com/archives/C033LCSFHF0)

### Event viewer: clarify “replay” functionality

The Replay button on an event row **does not replay only that row**. It **sets the selected bot’s read checkpoint to the chosen event** and **runs processing from there forward**, including **later events on the same queue**.

This was confusing and unspecific so we updated a few things to help make that clearer. 

#### Tool tip change on hover

We changed the tooltip that appeared when you hovered over the "replay" button to `Replay from this event` which is clearer than the original `Replay`

![Replay Tooltip](https://github.com/LeoPlatform/bus-ui/blob/changelog-screenshots/changelogs/screenshots/3.0.6-replay-tooltip.png?raw=true)


We also changed the Replay Event modal substantially to specifically callout the change in behavior:

![Replay Modal](https://github.com/LeoPlatform/bus-ui/blob/changelog-screenshots/changelogs/screenshots/3.0.6-replay-modal.png?raw=true)

This should address some of the confusion around what this functionality actually does.
